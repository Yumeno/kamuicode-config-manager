# AIモデルのリリース日調査ルール

このドキュメントは、kamuicode_model_memo.yamlに記載するAIモデルのリリース日を調査する際のルールを定義します。

## 調査の優先順位

### 1. クローズドモデル（商用API）
- **定義**: プロプライエタリなモデルで、APIを通じてのみアクセス可能
- **リリース日の定義**: モデルを開発元が**有償ユーザー向けに利用可能として提供開始した時期**
- **例**: OpenAI GPT-4、Google Gemini、Anthropic Claude など
- **調査先**:
  - 開発元の公式ブログ
  - 公式ドキュメント
  - プレスリリース
  - 公式X（Twitter）アカウント

### 2. オープンソースモデル
- **定義**: モデルウェイトが一般公開されているモデル
- **リリース日の定義**: モデルを開発元が**HuggingFace、Civitai、GitHub**などのリポジトリで公開し、一般に利用可能として提供開始した時期
- **注意事項**:
  - 派生の量子化モデル（GGUF、GPTQ等）がある場合は、**おおもとのモデルのリリース時点**を記録
  - WANなどのモデルで、大きなモデルサイズ・メジャーバージョン違いがある場合は留意すること
  - 例: WAN v2.2-5B と WAN v2.2-14B は異なるモデルとして扱う
- **調査先**:
  - HuggingFace Model Card
  - GitHub リポジトリのリリースタグ
  - Civitai モデルページ
  - arXiv 論文公開日（補助的に使用）

### 3. fal.ai / Replicate 独自提携モデル
- **定義**: 開発元が直接公開していないが、fal.aiやReplicateが独自に提携して提供しているモデル
- **リリース日の定義**: **fal.aiまたはReplicateがユーザーに利用可能として提供開始をアナウンスした時期**
- **調査先**:
  - fal.ai 公式ブログ: https://blog.fal.ai/
  - fal.ai Changelog
  - fal.ai 公式X（Twitter）アカウント
  - Replicate 公式ブログ
  - Replicate Changelog

### 4. Trainerモデル（LoRA、ファインチューニング）
- **定義**: 既存モデルをトレーニングするためのツール・機能
- **リリース日の定義**: **トレーニング対象のベースモデルの公開時期をそのまま踏襲**
- **例**:
  - `train-kamui-qwen-image-edit` → `Qwen Image Edit` モデルのリリース日と同じ
  - `train-kamui-flux-kontext` → `FLUX Kontext` モデルのリリース日と同じ

## 調査手順

1. **モデル名とserver_nameから開発元を特定**
   - server_nameのパターンから推測（例: `kamui-flux-*` → Black Forest Labs）
   - descriptionに記載されている開発元を確認

2. **モデルタイプを判別**（上記1〜4のいずれかに分類）

3. **該当する調査先で情報を検索**
   - 公式サイト、ブログ、GitHub、HuggingFace等
   - 日付形式は「YYYY年MM月DD日」または「YYYY年MM月頃」

4. **情報が見つからない場合**
   - 「調査困難」としてリスト化
   - 後でX（Twitter）でGrokを使用して調査

## 不明モデルの扱い

上記のルールに従って調査しても不明なものは、以下の情報を含むリストを作成:
- モデル名
- server_name
- 推測される開発元
- 調査で確認した情報源（あれば）
- 不明な理由

このリストはユーザーに提示し、X（Twitter）のGrokで追加調査を依頼する。

## 参考: よくある日付フォーマット

- **年月日が明確**: `2025年10月15日`
- **日が不明**: `2025年10月頃`
- **月が不明**: `2025年中頃`
- **年のみ**: `2025年`
- **完全に不明**: 空欄のまま、リストに追加
