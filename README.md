# **kamuicode Config Manager**

クリエイティブAIモデルのリモートmcpサーバー集”kamuicode”のサーバー構成を簡単に管理できるWebアプリケーションです。 複数のモデルから必要なものだけを選んで、フィルタリングされた設定ファイルを出力できます。

## **オンラインツールリスト (Webビューア)**

設定ファイルを編集するのではなく、単に利用可能なAIモデルやMCPツールの詳細（パラメータや使い方）を調べたい場合は、以下のオンラインビューアをご利用ください。インストール不要でブラウザからすぐに閲覧できます。

* **kamuicode List**: [https://yumeno.github.io/kamuicode-config-manager/](https://yumeno.github.io/kamuicode-config-manager/)

## **機能**

* **モデル構成の読み込み**: JSON形式のサーバー構成ファイルを読み込み
* **複数JSONファイルのマージ**: 主JSONと副JSON（複数可）を統合して管理
* **詳細情報の表示**: YAMLファイルからモデルの詳細情報（カテゴリ、正式名称、リリース日、説明）を表示
* **Web更新機能**: GitHubから最新のモデル定義を直接取得（ファイルダウンロード不要）
* **カスタムYAML対応**: 独自のモデル定義ファイルも使用可能
* **設定の永続化**: 選択したモードと設定は次回起動時に自動復元
* **柔軟なフィルタリング**: カテゴリやキーワードでモデルを絞り込み
* **多機能ソート**: カテゴリ、モデル名、リリース時期、ソースファイルで並び替え
* **チェック状態の管理**: 必要なモデルだけを選択し、その状態を保存・復元
* **設定JSONからのインポート**: 既存の設定JSONファイルを読み込み、含まれるサーバーのみを選択状態に
* **設定ファイルの出力**: 選択したモデルだけを含む新しいJSON設定ファイルを生成
* **MCPツールカタログ**: 各MCPサーバーが提供するツール情報（名前、説明、入力パラメータ）を閲覧
* **パスキー設定**: 出力JSONのパスキー（KAMUI-CODE-PASS）を環境変数プレースホルダー・直接入力・元の設定維持から選択
* **スクロール可能なモデル一覧**: 大量のモデルでも見やすいスクロール対応テーブル（ヘッダー固定）

## **ダウンロード**

[Releases ページ](https://github.com/Yumeno/kamuicode-config-manager/releases)から最新版のZIPをダウンロードしてください。

└── README.md                      \# このファイル

## **インストール**

1. ダウンロードしたZIPを任意のフォルダに解凍  
2. kamuicode-config-manager.html をブラウザで開く

\# Windowsの場合  
start kamuicode-config-manager.html

\# macOSの場合  
open kamuicode-config-manager.html

\# Linuxの場合  
xdg-open kamuicode-config-manager.html

または、ファイルエクスプローラーから kamuicode-config-manager.html をダブルクリックして開いてください。

## **使い方**

### **パスキー設定（KAMUI-CODE-PASS）**

Kamui Code MCPサーバーに接続するためのパスキーを設定します。この設定は出力JSONの `headers` 内 `KAMUI-CODE-PASS` エントリに反映されます。

| モード | 説明 |
|--------|------|
| **デフォルト設定** | 環境変数プレースホルダー `${KAMUI_CODE_PASS_KEY}` を全サーバーに設定（初期選択） |
| **パスキーを直接入力** | 発行されたパスキーを直接設定 |
| **変更しない** | 元のJSON設定をそのまま維持（headersを変更しない） |

#### **デフォルト設定（推奨）**

環境変数プレースホルダー `${KAMUI_CODE_PASS_KEY}` を使用します。Claude Desktopなどで環境変数を設定している場合に選択してください。

#### **パスキーを直接入力**

発行されたパスキーを直接設定する場合に選択します。

1. **「パスキーを直接入力」** を選択
2. 入力欄にパスキーを入力（パスワード形式で非表示）
3. 目のアイコンをクリックすると入力内容を確認できます
4. **「保存」** ボタンをクリックしてブラウザに保存

#### **変更しない**

元のJSON設定をそのまま維持したい場合に選択します。`headers` の内容は変更されません。

**注意**: パスキーはブラウザのLocalStorageに保存されます。共有PCでの使用にはご注意ください。

### **ステップ1: JSONファイルの読み込み（必須）**

#### **主JSONの読み込み（必須）**

1. **「1. mcpServers (.json)」** エリアの **「主 (Main) JSON」** ファイル選択ボタンをクリック
2. メインで使用したいmcp設定JSONファイルを選択
3. 読み込みが成功すると、すべてのモデルが一覧表示されます

#### **副JSONの読み込み（任意・複数可）**

分割個別提供されているmcp設定jsonファイルを必要に応じて追加して、一緒に取り扱うことができます。

1. **「副 (Secondary) JSON」** のファイル選択ボタンをクリック
2. 追加の設定ファイルを選択（複数選択可能）
3. 読み込み済みファイルが一覧表示されます

#### **マージ設定（競合解決ルール）**

副JSONを読み込むと、マージ設定が表示されます。同一のserver_nameが両方に存在する場合の挙動を選択できます。

| モード | 説明 |
|--------|------|
| **主を優先 (Default)** | 重複キーは主の設定を使用し、副は無視 |
| **副を優先 (Override)** | 重複キーは副の設定で上書き |
| **両方を含める (Keep Both)** | 重複キーは両方表示（副にサフィックス付与） |

**ソース列**: 副JSONを読み込むと、テーブルに「ソース」列が表示され、各モデルがどのファイルから来たかが色分けバッジで表示されます（青：主、緑：副）。

**JSON形式の例：**

{
  "mcpServers": {
    "file-upload-kamui-fal": {
      "type": "http",
      "url": "\[https://example.com/\](https://example.com/)...",
      "description": "File Upload to FAL..."
    },
    "t2i-kamui-dreamina-v31": {
      "type": "http",
      "url": "\[https://example.com/t2i/\](https://example.com/t2i/)...",
      "description": "Bytedance Dreamina v3.1..."
    }
  }
}

### **ステップ2: モデル情報の読み込み（推奨）**

モデルの詳細情報（カテゴリ、正式名称、リリース日、特徴）を読み込むことで、一覧がより分かりやすくなります。

#### **標準モデル情報を使用（推奨）**

GitHubリポジトリから最新のモデル定義を直接取得します。

1. **「標準モデル情報を使用」** を選択（デフォルト）  
2. **「モデル情報を読み込む」** ボタンをクリック  
3. インターネット経由で最新のYAMLが取得され、自動的に適用されます  
4. 読み込み後は **「最新に更新」** ボタンで再取得できます

#### **カスタムYAMLファイルを使用**

独自に編集したYAMLファイルや、ローカルに保存したファイルを使用します。

1. **「カスタムYAMLファイルを使用」** を選択  
2. ファイル選択ボタンから任意の .yaml または .yml ファイルを選択  
3. 選択したファイルの内容が適用されます

**YAML形式の例：**

ai\_models:  
  text\_to\_image:  
    \- name: ByteDance Dreamina v3.1  
      server\_name: t2i-kamui-dreamina-v31  
      release\_date: 2024年12月  
      features: "(ByteDance) 高品質なテキストからの画像生成。"

**注意**: features にコロン（:）が含まれる場合は、文字列全体をダブルクォーテーション（"）で囲んでください。

### **ステップ3: フィルタとソート**

モデル一覧から必要なものを見つけやすくします。

#### **カテゴリで絞り込み**

* ドロップダウンメニューから特定のカテゴリを選択  
* 例: text\_to\_video を選択すると、そのカテゴリのモデルのみ表示

#### **テキストで検索**

* 検索ボックスにキーワードを入力  
* モデル名、server\_name、解説のいずれかに部分一致するモデルを表示  
* 例: 「Vidu」と入力すると、Viduを含むモデルのみ表示

#### **ソート（並び替え）**

以下のテーブルヘッダーをクリックすることでソートできます：

| 列 | 説明 |
|----|------|
| **カテゴリ** | カテゴリ名でソート |
| **モデル名** | モデル名でソート |
| **リリース時期** | リリース日でソート |
| **ソース** | ソースファイル名でソート（副JSON読み込み時のみ表示） |

* クリックごとに **昇順（▲）→ 降順（▼）→ 解除（元の順序）** と切り替わります
* **「ソート解除 (JSON順に戻す)」** ボタン：ソート中に表示され、クリックで元のJSON順序に戻します

### **ステップ4: チェック状態の管理（便利機能）**

複数のプロジェクトで異なるモデルセットを使い分ける場合に便利です。

#### **チェックボックスの操作**

* **個別選択**: 各モデルのチェックボックスをON/OFF
* **一括操作**:
  * 「フィルタ内すべてチェック」: 現在表示中のモデルをすべて選択
  * 「フィルタ内すべて解除」: 現在表示中のモデルをすべて解除

#### **設定JSONからインポート**

既存の設定JSONファイルから選択状態をインポートできます。これにより、過去に出力した設定ファイルや、他のユーザーから共有された設定に合わせて素早くモデルを選択できます。

1. **「設定JSONからインポート」** ボタンをクリック
2. インポートしたい設定JSONファイルを選択
3. そのJSONに含まれるMCPサーバーのみがチェック状態になり、含まれないサーバーはチェックが外れます

**用途例:**
* 以前に出力した設定ファイルと同じ構成を再現したい場合
* 他のユーザーの設定を自分の環境にインポートしたい場合
* 複数の設定パターンを切り替えて使いたい場合

#### **チェック状態の保存と読み込み（YAML形式）**

* **状態を保存**: 現在のチェック状態を mcp-check-state.yaml としてダウンロード
* **状態を読み込む**: 過去に保存したYAMLファイルを読み込んでチェック状態を復元
  * **注意**: YAMLに記載がないモデルは、デフォルトで true（チェックあり）になります
* **クリア**: 読み込んだ状態をリセットし、すべて true に戻します

**チェック状態YAML形式の例：**

file-upload-kamui-fal: true
t2i-kamui-dreamina-v31: true
t2i-kamui-flux-krea-lora: false

### **ステップ5: 最終JSONの出力**

1. **「4. 最終JSONを出力」** エリアの「出力JSONファイル名」を入力  
   * デフォルト: mcp-kamui-code-filtered  
2. **「JSONを生成して保存」** ボタンをクリック  
3. チェックがONのモデルだけを含むJSONファイルがダウンロードされます  
   * 元のmcpServers形式と同じ構造で出力されます

### **MCPツールカタログの利用（任意）**

各MCPサーバーが提供するツールの詳細情報を確認できます。

#### **ツールカタログの読み込み**

1. モデル一覧のヘッダーにある **「ツールカタログ読込」** ボタンをクリック  
2. GitHubから最新のカタログ（mcp\_tool\_catalog.yaml）が自動取得されます  
3. 読み込み完了後、各モデル行の「ツール詳細」列にボタンが表示されます

#### **ツール詳細の確認**

1. **「詳細 (N)」** ボタンをクリック（Nはツール数）  
   * カタログに該当サーバーがない場合は「N/A」と表示されます  
2. ダイアログが開き、そのサーバーが提供するツール一覧が表示されます  
3. 各ツールについて以下の情報を確認できます：  
   * **ツール名**: 関数名（submit / status / result の順で表示）  
   * **説明**: ツールの機能説明  
   * **入力パラメータ**: パラメータ名、型、必須/任意、説明

#### **ダイアログの操作**

* **ESCキー** または **オーバーレイをクリック** でダイアログを閉じます  
* ダイアログ内はスクロール可能です

**Note**: ツールカタログは定期的に更新されます。最新情報を取得するには「カタログクリア」後に再度読み込んでください。

## **トラブルシューティング**

### **YAMLファイルが正しく読み込めない**

* features フィールドにコロン（:）が含まれている場合、文字列全体をダブルクォーテーション（"）で囲んでください  
* YAML形式が正しいか確認してください（インデントはスペース2個）

### **JSONファイルが読み込めない**

* ファイルが正しいJSON形式か確認してください  
* mcpServers キーがトップレベルに存在するか確認してください

### **Web更新機能が動作しない**

* インターネット接続を確認してください  
* ブラウザのキャッシュをクリアして再試行してください  
* 「カスタムYAMLファイルを使用」モードに切り替えて、ローカルファイルを使用してください
